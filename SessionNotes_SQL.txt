1) between	2) in(list)	3)is null	4) like

select *from emp where sal not between 1500 and 3000;

When ever we use two tables in oracle query it will perform carteisian product

To eliminate this concept we can use a concept of joins.

Joins are classified into different types basically three types.

1) Simple Joins
2) Self Joins
3) Outer Joins

Join is a condition or predicate used in a query statement to project the required rows from the cartesian product.
Simple Joins:

A simple join is a condition used to project all the associated rows from the cartesian product.

Equi Join: When a join condition is based on the relational operator = is called Equi join

Non Equi Join: When a join condition is based on the relational operators other than equal 
to operator when it is called a non equi join

Self Join:
When a relation or table is joined to itself then it is called as the self join.

Outer join:

When a join condition is used to project all the associated and also non associated 
rows from the cartesian product then it is called an outer join.
